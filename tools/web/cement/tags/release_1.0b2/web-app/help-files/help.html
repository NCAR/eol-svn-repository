Uploading Metadata and Data for AON Projects

An on-line interface has been established to permit the upload of metadata and data. The url is: www.eol.car.edu/projects/aon-cadis. The upload site can be accessed via the links list (in gray) along the left side of the page or directly at http://cdp.ucar.edu/cadis .

Obtaining a CADIS User Name and Password

Accounts have been set up on the CADIS site for lead PIs of projects included in CADIS. Project PIs can expect an email from CADIS support staff with user name and initial passwords, and instructions on changing passwords. Project data areas have been set up, as well, under the SEARCH project categories. PIs may designate "metadata contacts" who may create and edit metadata for datasets, upload data to datasets, and handle other administrative functions related to making data available on the CADIS portal. Detailed instructions on designating "metadata contacts" will be in the initial email. Please contact CADIS support (cadis@eol.ucar.edu) if you need further instructions for accessing the portal.

Logging On and Navigating

Connect to the log-in page and provide user name and password. Once you are logged in links will appear on the left side of the page.

You can enter metadata without uploading data (metadata will include a field that says where the data are) or, you can upload both metadata and data. You must provide a minimum set of required metadata before uploading a data file into CADIS.

Click on one of the links under Metadata for Datasets to enter the metadata entry form.

Entering metadata

This section includes a description of how to fill out all the metadata fields for a data set using the CADIS metadata authoring interface. Before beginning, decide what collections of data will constitute individual data sets. You will be filling out metadata for each dataset submitted within a given AON project.

Please note that you must fill out the required fields (in red) on the form in order to upload a dataset. You can cancel at any time but all of your work is erased and you will start over. A better idea may be to fill the required fields with anything, then save the metadata. You can then retrieve it later and continue working.

A quick way to enter metadata is to use an existing metadata record as a template. You can use any of the metadata records that you have access to for this purpose. To do this, choose a dataset from the list of datasets previously submitted. Edit the record as needed, changing the title of the dataset as a minumum. Then scroll to the bottom and click on "Create metadata for a new dataset using this one as a template".

The metadata fields, taken together, make up the CADIS metadata profile. CADIS metadata are compatible with (that is, they can be mapped to) other medata standards including the NASA Global Change Master Directory's (GCMD) Directory Interchange Format (DIF) and the International Polar Year metadata profile. The CADIS metadata is most like DIF metadata, with a few extra fields. Many DIF and CADIS fields are identical, and we have drawn heavily on the pioneering work of the GCMD project for the CADIS profile and this metadata authoring tool. The Directory Interchange Format (DIF) Writer's Guide has information and examples for many fields that you may find useful.

Item 1. Project. Choose from the drop down list of NSF funded projects. Choose the project that funds your data collection, or that your data collection is in support of. Please contact Jim Moore or Florence Fetterer if you don't see your project or if you have questions. If there is an error in the project name please contact CADIS support (cadis@eol.ucar.edu) to correct the problem.

Item 2. Dataset Title. Please make this dataset title descriptive of the content. Put the most important words first, and do not include dates. Acronyms are OK. If the title doesn't fit in the entry window it is probably too long. Keep in mind that like the title of a research paper, the title of a data set may be used in a citation.

Item 3. Dataset Summary. This is a paragraph that describes the 'who, what, where, when, and why' of the dataset you are submitting with this metadata. It will appear in the online AON data catalog. Web crawlers will index this paragraph, and as a result the description provided here will assist others who are searching for data using a search engine and keyword search. This field can be typed in or simply cut and pasted from other existing information such as the summary from your NSF proposal (but remember, it should describe the dataset, not the project as a whole).

Items 4 and 5. Begin and End date. Choose from the drop down menus to provide the temporal coverage of the data set. If hour and day of month are known, please include them, however, it is not essential.

Items 6-9. Minimum and Maximum Latitude and Longitude. Please supply the values for spatial coverage that describe a box around your data set in term of minimum and maximum latitude and longitude. If the data are at one geographical location only, enter the latitude and longitude of that point. For a moving platform, use a box that bounds the track of the platform. It is not necessary to be extremely precise if the information is not readily at hand, position to a tenth of a degree or even a degree is usually good enough. The following guidelines from the GCMD may be useful.

Southermost_Latitude: The southernmost geographic latitude covered by the data. From: 0 - 90 deg North or 0 - 90 deg South. For example, 60

Northernmost_Latitude: The northernmost geographic latitude covered by the data. From: 0 - 90 deg North or 0 - 90 deg South.

Westernmost_Longitude: The westernmost geographic longitude covered by the data. From: 0 - 180 deg East or 0 - 180 deg West. The Prime Meridian is 0 degrees, measured positive (+) eastwards of the PM. For example, 45 deg W.

Easternmost_Longitude: The easternmost geographic longitude covered by the data. From: 0 - 180 deg east or 0 - 180 deg west. The Prime Meridian is 0 degrees, measured positive (+) eastwards of the PM.

Item 10. Location Keyword. Choose the one that fits best.

Item 11. Discipline. Choose the one that fits best. These are used to group AON datasets at a high level.

Item 12. Frequency. Choose from the drop down menu to indicate the temporal resolution of the dataset. When there is more than one choice, choose the frequency that best approximates the sampling frequency for your data

Item 13. Spatial Type. Choose the data type that most closely matches your data.

Item 14. Resolution.Choose from the ranges given by the GCMD keywords. For some data sets (gridded satellite image data, for example), resolution can be harder to define. Just choose a resolution that best describes the data. You can include more detailed information in documentation you provide.

Item 15. Platform Keyword. Choose the platform or platforms for the instruments that acquired the data.

Item 16. Instrument Name. Choose from the list. The last choice is "other" if the instrument is not in the list.

Item 17. Science Keyword. Choose from the NASA Global Change Master Directory list of topic areas or parameters.

Item 18. ISO Topic. Select one of the topic categories. Using these helps your data set's metadata to be ISO compliant. This may be useful in the future for international data sharing.

Item 19. Metadata Contact. The drop down menu includes PIs for the projects. You can enter a different name if needed. This is person who is responsible for the content of the metadata record. Generally, this will be the person filling out the form. If the responsibility shifts from the original author to another person (e.g. from an investigator to a data manager), the field should be updated to the new responsible person. This is so if a question comes up about the metadata, we know who to ask.

Item 20. Data Center Contact. The person (or Help Desk, or data center itself) who is responsible for the distribution of the data. If data are submitted to CADIS, this will be Don Stott. If data are kept elsewhere, and only metadata are submitted to CADIS, it will be the person responsible for the data.

Item 21. Data Access. A description of how to get to your data. If you submit data to CADIS, leave this blank. Otherwise, this will usually be a link to a page that conveys both how and where to get the data.

Item 22. Distribution Format. Remember that ASCII is not a format in the metadata sense. You need to be more specific. This is a required field.

Item 23. Progress. This is meant to describe the state of the data being submitted where Planned refers to data sets to be collected in the future and are thus unavailable at the present time, in-work means preliminary or on-going, and complete means final, that is, data sets in which no updates or further data collection will be made.

Item 24. Data Tags. These are other unique identifiers. This allows you to add other unique tags (e.g. more specific discipline, project related IOP or station number) or other remarks on your datasets that may help in locating and using these data.

Items 25. & 26. Related Resources References (citations) to work that uses the data, and link to full product documentation go here. Users will insert a URL or citation, then tag it with what category it falls in using dropdowns.

Item 27. Dataset Language. If the dataset (data and/or documentation) is in a language other than English, please enter the language.

Item 28. Access Restrictions. We have assumed that all AON datasets are unrestricted in access once they are uploaded to the CADIS data server. If you do not want such open access, you must speak with the NSF manager about exceptions. These may include any special restrictions, legal prerequisites, limitations and/or warnings on obtaining the data set. Additional information and examples from GCMD.

Item 29. Use Constraints. Here one may describe how the data may or may not be used after access is granted to assure the protection of privacy or intellectual property. This includes any special restrictions, legal prerequisites, terms and conditions, and/or limitations on using the data set. Data providers may request acknowledgement of the data from users and claim no responsibility for quality and completeness of data. Additional information from GCMD.

Preview Page

After you have completed the 29 item metadata entry form, click the "Save metadata for new dataset" button at the bottom of the page. If you started with a previously submitted metadata record, use the "Create a new dataset using this as a template" button. CADIS now reviews your metadata entry for completeness. If you have made an error or omission, you are alerted to this with a red box at the top of the form highlighting problem areas as well as a red box around the specific item in error. Please correct them at this time. Click on the Submit metadata for a dataset button once more to continue the submission preview.

You will be brought to the Review Dataset Metadata page where you now review your complete metadata in a list form. If you are satisfied, click the Submit this dataset to CADIS button on the left or at the bottom of the page. This may take a moment. Remember, it is possible for you to edit the metadata at any time during this session or by re-entering the CADIS data/metadata upload site.

Uploading Data

After submitting the metadata you will see a confirmation page that includes next steps along the left side labeled "Actions for the Dataset". As noted above, you may edit the metadata using the link. You will see the same forms as before.
