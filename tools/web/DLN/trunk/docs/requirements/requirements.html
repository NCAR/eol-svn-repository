Required Views

  • List of all projects
    • Displays the project id, dataset prefix, and active flag
    • Allows for changing the activation of one of the projects
  • List of datasets for a project
    • Displays all datasets associated with the project (even if the dataset has not been entered in the DTS database).
    • Displays the dataset id, title, type, task contacts
    • Displays flags for the status of each task, for the readme flag, and if the dataset is in the master list for the project.
    • Allows for opening the editor for a dataset.
    • Allows for sorting by the dataset id, title, type, task contact.
    • Has a filter for displaying resolved/not resolved datasets for each status.
    • Has a filter for displaying documented/not documented datasets.
    • Has a filter for displaying dataset in/not in the master list.
    • Has a filter for displaying datasets with a specififed dataset type.
  • List of datasets for a user (by task [ingest,load,approve])
    • Displays the dataset id, title, type, project list, and contacts for the unselected tasks.
    • Displays flags for the status of each task, for the readme flag, and if the dataset is in the master list for each project.
    • Allows for opening the editor for a dataset.
    • Allows for sorting by the dataset id, title, type, project, or task contact.
    • Has the same filters as the List of datasets for a project view.
  • View of dataset information
    • Displays the dataset id, title, project list, dataset type, entry date and contact, ingest location, data to archive location, archived location, external contact, internal contact, ingest contact, ingest status, load contact, load status, approve contact, approve status, list of remote URLS, readme flag, master list flag for each project, and the notes for the dataset.
    • Allows for the moving the previous or next entry in the list of datasets.
    • Allows for opening the editor for the dataset.

Required Functionality

  • General Functionality
    • Dataset ids must be unique as defined by the data archive system.
    • All changes to a dataset must be applied to all projects associated with the dataset.
  • Activate/Deactivate a Project
    • Changes the active flag in the DTS database to make the project appear/disappear from project lists in the DTS.
  • Activate/Deactivate a User
    • Changes the active flag in the DTS database to make the user appear/disappear from selection lists in the DTS.
  • Create a new Dataset
    • Defines the following fields for a dataset (all fields are in the DTS database unless specified otherwise):
      • Dataset ID - Unique id to define the database access for the dataset (in data archive system)
      • Title - The displayed name of the dataset (in data archive system)
      • Project List - The list of projects associated with the dataset (in data archive system)
      • Dataset Type - The type of dataset (source, processed, ...)
      • Internal Contact - The internal contact for questions on the dataset (in data archive system)
      • Entry Date - The date the dataset was entered into the DTS system.
      • Entry Contact - The user who entered the dataset into the DTS system.
      • Readme Flag - A flag that specifies if one or more documents exist for the dataset.
      • General Dataset Notes - The set of notes for the dataset that are not associated with a specific task.
      • Ingest Contact - The person who is in charge of ingesting the data for the dataset.
      • Ingest Status - The status of the ingest task.
      • Ingest Location - The directory path on the system where the ingested data was placed.
      • Ingest Notes - The set of notes that are for the ingest task.
      • Load Contact - The person who is in charge of loading the data on the data archive system.
      • Load Status - The status of the load task.
      • Data to Archive Location - The directory path of the data to be loaded into the data archive system.
      • Archive Location - The directory path where the final archived data is stored for the data archive system.
      • Load Notes - The set of notes that are for the load task.
      • Approve Contact - The person who is in charge of approving the final state of the dataset.
      • Approve Status - The status of the approve task.
      • Approve Notes - The set of notes that are for the approve task.
    • Inserts values into the data archive system database:
      • Defaults the lat/lon box to the maximum sized box defined by the projects selected for the dataset.
      • Defaults the start/end date to the maximum date range defined by the projects selected for the dataset.
      • Defaults the hidden flag to true
      • Assigns the dataset_id, name, and internal contact from the values entered in the form.
      • Associates the dataset to the selected project(s).
    • Inserts values into the DTS database:
      • Enters the values not in the data archive system into the DTS database.
      • Creates new notes and associates them with the dataset.
      • Only creates entries in tables that have values. (i.e. If a dataset is not loaded it won't create a load entry in the database.)
  • Clone a Dataset
    • Initializes the dataset fields from the source dataset for the clone except for the dataset id (which must be a new unique value).
    • Inserts the values into the data archive system database and the DTS database like in Create a new Dataset section above.
  • Edit a Dataset
    • Defines the same fields as listed in the Create a new Dataset section above.
    • Cannot change the dataset id for the dataset.
    • Updates the values in the data archive system database.
    • Updates the values in the DTS database
  • Delete a Dataset
    • Completely removes the dataset from both the DTS database and the data archive system database.

Purposely Omitted Functionality

  • Adding, Editing, Deleting Projects (This is done in other tools)
  • Adding, Editing, Deleting Users (This is done in other tools)
  • Adding, Editing, Deleting Statuses (This is done on the command line at this time)
  • Adding, Editing, Deleting Ingest Types (This is done on the command line at this time)
