Query Master List: How To

Step 1:

Specify the url of the script in the CGI bin followed by a question mark (?): http://www.joss.ucar.edu/cgi-bin/dpg/arctic_ml/master_query?

Step 2:

Specify the name of the project using the parameter 'project', this must be set or you will get an error:
http://www.joss.ucar.edu/cgi-bin/dpg/arctic_ml/master_query?project=SBI

Step 3:

Narrow the search by specifying values for different fields in the databaase. Possible fields are:

-platforms
the name(s) of the platform(s) to pull from database, if not set all platforms are shown

-disciplines
the name(s) of the discipline(s) to pull from database, if not set all disciplines are shown

-data_types
the name(s) of the data type(s) to pull from database, if not set all data types are shown

-sites
the name(s) of the site(s) to pull from database, if not set all sites are shown

-spat_resolutions
the name(s) of the spat_res(s) to pull from database, if not set all spatial resolutions are shown

-year
the year to pull from the database. Pulls all datasets whose start_year and end_year fall around the year given.

-comments
the value(s) of the comments field to pull from the database.

-phases
the value(s) of the phase field to pull from the database.
Note: More than one comment string or one phase string can be entered.

-methods_of_obs
the value(s) of the method_of_obs (Method of Observation) field to pull from the database.

-dateeq, datelt, dategt, datele, datege
conditionals to pull datasets submitted to JOSS from a given time period. Date must in the YYYY-MM-DD format.
Ex: http://www.joss.ucar.edu/cgi-bin/dpg/arctic_ml/master_query?project=ATLAS&fields=title&fields=storm_id&fields=date&dategt=2002-09-30

Step 4:

Determine the display by specifying the fields to display, the sort order, and a title.

-fields
the field(s) to display. The order given is the order they appear from left to right.
Possible fields:
date data_type documentation comments frequency (observing frequency from CODIAC)
storm_id discipline phase year (from CODIAC) begin_date (from CODIAC)
title platform url start_year (from CODIAC) end_date (from CODIAC)
author site doc_url end_year (from CODIAC) size (sum of all the file sizes in CODIAC)

Note: if year is specified start_year 'dash' (-) end_year is displayed (1999-2000 for example).

-order
the field(s) by which to sort the table.
Possible order:
date data_type phase
storm_id discipline comments
title platform url
author site doc_url
Currently, the table cannot be sorted by any of the fields pulled from CODIAC. If this is desired the query script will have to be updated.
Example: &order=date&order=storm_id - in this case the table is sorted first by date and then by storm_id.

-title
the title to display at the top of the table.

Note: The url cannot contain spaces! Spaces are replaced with plus signs (+). Example: &title=This+is+a+title+with+spaces

display parameter

The display parameter allows you to pull default tables and summary information from the database. Along with the value of the display parameter the project parameter must be included and the title parameter may also be included.

-display
Possible values:
-summary - displays a summary of the datasets in the database for the given project.

Examples:

http://www.joss.ucar.edu/cgi-bin/dpg/arctic_ml/master_query?project=ATLAS&display=summary

http://www.joss.ucar.edu/cgi-bin/dpg/arctic_ml/master_query?project=SBI&fields=date&fields=title&fields=author

http://www.joss.ucar.edu/cgi-bin/dpg/arctic_ml/master_query?project=SBI&fields=date&fields=platform&fields=title&fields=author&platforms=HLY-02-03&title=Platform:+HLY-02-03+Datasets

http://www.joss.ucar.edu/cgi-bin/dpg/arctic_ml/master_query?project=SBI&fields=date&fields=discipline&fields=title&fields=author&disciplines=Ice&disciplines=Water&order=discipline&order=title

http://www.joss.ucar.edu/cgi-bin/dpg/arctic_ml/master_query?project=SBI&fields=title&fields=author&order=author&order=title&title=This+is+the+last+example
