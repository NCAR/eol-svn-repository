General Master List (Stand Alone) Requirements

Required Information

Project

  • Project ID
  • Display Name for the Project
  • Length (in days) for how long a dataset is considered "new"
  • Path/URL for the viewer
  • Set of allowed Categories (some could be hidden)
  • Set of Datasets (some could be hidden)

Category

  • Category Id
  • Display Name
  • Optional parent category for category heirarchy

Dataset

  • Dataset Id
  • Dataset Title
  • Dataset URL
  • Documentation URL
  • Expected Date
  • Entered Date
  • Update Flag
  • Hide Flag (per project)
  • In Progress Flag (per project)
  • Set of Projects
  • Set of Categories

Required Views

  • List of all projects with a Master List
    • Displays the project id and project name for all projects with a Master List
  • List of all categories
    • Displays all known categories in the Master List
  • Project List
    • Lists all categories with all associated datasets for the project.
    • Shows all hidden datasets and categories

Required Functionality

  • General Functions
    • Page caching must be turned off to prevent invalid states
    • Dates must be checked for validity
    • Dataset ids must be unique
    • Any change to the database must regenerate all related static HTML pages for all affected projects.
    • In Progress datasets are only for the selected project, but updates and dates are applied to all projects.
    • A dataset can be hidden for a project and will hide it for all associated categories.
    • A category can be hidden for a project hiding all associated datasets for the category.
  • Adding a Project
    • Defines the path for the ML, the display name and its id
    • Assigns the set of categories for the project
    • Generates the initial static HTML pages for the view
  • Cloning a Project
    • Defines the path for the ML, the display name and its id
    • Removes unnecessary cloned categories
    • Adds necessary cloned categories
    • Remvoes unnecessary cloned datasets that do not have a category assigned in the project
    • Removes other unnecessary cloned datasets
    • Adds associations for datasets that keep the same id for the project
    • Determines new dataset ids for datasets that are created from the source project
    • Generates the initial static HTML pages for the view
  • Editing a Project
    • Can change the path for the ML and display name
    • Can change the set of categories associated with the project.
    • Regenerates the initial static HTML pages for the view
  • Delete a Project
    • Completely removes all datasets that are only associated with the project
    • Removes all associates to dataset for the project
    • Remvoes all associates to categories for the project
    • Removes all generated pages for the view
    • Removes the project itself from the database (not in jedi)
  • Create a Category
    • Autogenerates id
    • User defines a unique name, and optionally a parent
  • Editing a Catetory
    • Changes the name, but must remain unique
    • Regenerates static HTML pages for all projects that are associated to the category
  • Deleting a Category
    • Can only be done to categories that do not have any project associations (can have dataset associations that will be removed)
    • Removes the category from the database
    • Static HTML pages do not need to be regenerated because there are no project associations for the category
  • Adding an Existing Dataset to a Project
    • Add an association between the dataset and project
    • Set the in progress and update flags to false (0)
    • The dates remain unchanged
    • Regenerates the static HTML pages for the project
  • Adding a new Dataset to a Project
    • Requires dataset title
    • Autogenerates a dataset id if not assigned
    • Flags are set to false (0) as defaults
    • Dates are set to NULL as defaults
    • Dataset must be assigned to at least 1 category
    • Dataset is assigned to the project
    • Regenerates the static HTML pages for the project
  • Editing a Dataset
    • Does not allow for changing the project associations
    • Can change the categories associated with the dataset
    • Can only change the general dataset information and specific information for the current project
    • Regenerates the static HTML pages for all associated project for the dataset
  • Deleting a Dataset from a Project
    • Deletes the association with the project
    • Deletes the entire dataset if it is no longer associated with a project
    • Regenerates the static HTML pages for the project

Purposely Omitted Functionality

  • A dynamic viewer - all changes will write static HTML files to the specified directory for the project
