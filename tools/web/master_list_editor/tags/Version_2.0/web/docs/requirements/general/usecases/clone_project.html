Use Case: Cloning a Project

  1. The user wants to create a new project Zeta from an existing project Gamma.
  2. The user goes to the Master List home page and selects Clone a Project.
  3. The tool displays the list of projects that can be cloned and the list of current projects in progress of being cloned.
  4. The user chooses project Gamma as the source project.
  5. The tool displays the project form with the values filled in for project Gamma.
  6. The user updates the fields in the form for Zeta and clicks on the Next button.
  7. The tool displays the category selection page.
  8. The user updates the selection of categories that are to be used in Zeta and selects the Next button.
  9. The tool displays the list of datasets that are in Gamma and have a category that has been selected for Zeta.
  10. For each dataset in the list the user marks the dataset and the tool changes the background color:
    • The user marks the dataset as Discard and the tool draws a line through the entire entry.
    • The user marks the dataset as Keep with new Id and the tool draws a line through the dataset id and changes the text color.
    • The user marks the dataset as Keep with same Id and the changes the text color.
  11. The user clicks on the Finished button.
  12. The tool creates project Zeta in the database.
  13. The tool creates the associations for the selected categories with Zeta.
  14. The tool associates all datasets that were marked with Keep with same Id with project Zeta.
  15. The tool creates new datasets for the datasets that were marked with Keep with new Id and automatically generate an id with no flags set and no dates defined.
  16. The tool associates the datasets marked as Keep with new Id with the categories that are associated with the source dataset.
  17. The tool associated the datasets marked as Keep with new Id with project Zeta.
  18. The tool generates the static HTML pages for Zeta.
  19. The tool transfers control to the editor for Zeta.
