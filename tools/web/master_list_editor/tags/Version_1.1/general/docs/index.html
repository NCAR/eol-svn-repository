CGI Scripts

Actual Location: /web/cgi-bin/master

This directory contains a sub-directory for each project using the master list. Within each project directory is a copy of the master_list script which allows users to edit the project's master list. The script is essentially the same for each project with a few changes to constant variables at the top.

Problems/Issues

'New!' Icon:
In May, 2004 a 'new' icon feature was implemented such that if a dataset was added to the system within the last 90 days a 'New!' icon is placed above the date. Since the public side of the ML system is not dynamic - the public views a static html file as opposed to some CGI script which queries the database - these 'New!' icons can become out-of-date if the master list is not updated for a long period of time. Each time a project's master list is updated the script writes a new static html file for the public. When this file is written it checks each dataset to see if it requires a 'New!' icon. So only one dataset has to be updated to remove any out-of-date 'New!' icons.

Does not support multiple projects
This system was originally written for IHOP_2002 and not for use by multiple projects. Therefore each time a new project is added a new set of tables has to be created and a new version of the script has to be set up. The Improved Master List was developed to get rid of all the down-falls of this system but was never put into use.
