CGI Code - Editor and Query

Directory: cgi
Actual Location: /web/cgi-bin/dpg/arctic_ml

This directory contains the CGI code, written in Perl, to display the editor and to query the database for view by the public.

Editor:

master_maint - displays all aspects of the editor.
lib/ - contains various modules used by both editor and query scripts.
mysql_codiac/ - contains new scripts for when CODIAC goes to MySQL see other

Query:
The master_query script was set up to give Greg flexability in how the project master lists are displayed. The script takes several arguments including what parameters to display, what to sort by, what title to give the list, etc. The usage of this script is outlined in the Query How To.

master_query - queries the database and displays the master list
ml_summary - simple script to display a summary of all the datasets in a particular project. usage: ml_summary?project=ATLAS
codiac_readme - if no 'Documentation' link is entered into the master list, the master_query script displays a link to the codiac_readme script which redirects the user to the dataset's readme file in CODIAC. usage: codiac_readme?storm_id=46.001
lib/ - contains various modules used by master_query

Database

The Arctic Master List uses the master_list database located on thunder. The CGI scripts use the suldan username/password to access the database (talk to Greg or Mark to get the suldan password). Other users may be able to access the database (I think Don has the permissions) but talk to Mark about those kinds of details. The version of MySQL on thunder is rather old and lacks Foreign Key Constraints, which are used by the database. These constraints are maintained by the application, so should not be a problem, but watch out when adding new records manually.

To access the database:
mysql -h thunder -u suldan -p master_list

Problems/Issues



Other

CODIAC to MySQL The directory /web/cgi-bin/dpg/arctic_ml/mysql_codiac contains new scripts which can be used when CODIAC moves over to MySQL. Currently it queries the injedi3 database on gale. The name and host of the database will probably change so the scripts will have to be updated slightly. See the scripts themselves to see how to change the database and host names. The only scripts which are affected is the master_query script and codiac_readme script. Just copies those over to the /web/cgi-bin/dpg/arctic_ml directory.

Cascading Style Sheets (CSS):
The master_query script uses a simple cascading sheet to display the master lists. Each project has its own style sheet to match Greg's main project page. The css files are located here: /web/docs/atlas/css. ALL css files are located in this directory, including SBI and SHEBA. At one point they were located somewhere under /web/docs/dpg which caused the dpg username/password to pop up. Placing them in the atlas area was the quick fix, they should probably be moved to a more logical place. The file names follow the convention - PROJECT.css - where project is the project name entered into the master list database.
