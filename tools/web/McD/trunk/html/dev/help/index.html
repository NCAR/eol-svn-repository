Data Loading Notes
Viewing Dataset Information
'Listing' View
The 'Listing' View of the Data Loading Notes displays basic information about the datasets associated with a particular Project, Loader or Checker.
Editing Clicking the icon will pull up the edit window to edit the corresponding dataset.
Hide/Show
Selecting an item from the Hide drop down box will hide all datasets with the corresponding attribute (Documented, Checked, Loaded etc.) from view. To bring the hidden datasets back into view select the same attribute from the Show drop down box. Multiple attributes can be hidden, selecting Show All from Show will bring all datasets back into view.
Sorting
Click the name of the column to sort by Date, Title, Storm_id etc.
Status Icons
  • - Dataset is a link to a remote website, or page outside of CODIAC.
  • - Dataset has a completed 'Readme' file.
  • - Dataset has been loaded into CODIAC
  • - The entry in CODIAC has been checked by the dataset checker.
'Detailed' View
The 'Detailed' View displays all Data Loading information for a particular dataset. This view is obtained by clicking on the dataset title in the 'Listing' View'.
Navigation The    icons allow you to navigate back and forth between datasets in the Project, Loader or Checker list.
QCODIAC Clicking on the QCODIAC link opens a new window with the QCODIAC results for the current dataset. Displays the dataset description, on line physical directory listing, and off line physical directory listing in CODIAC. QCODIAC Home
Add/Edit Dataset Information
Storm Id/Next Id In the Storm Id text field enter the CODIAC storm_id for the dataset. If the dataset does not have storm_id simply leave the field blank, do NOT enter 99.999, doing so may cause problems when viewing the public side of the Master Data Set List. Clicking on the Next Id link will change the value in the text field to the next available storm_id. This is merely a suggestion of what should be used, not mandatory. Note: The Next Id link does not allows choose an available id, this is a known bug that will be fixed in the near future.
Ingest/Archive Location Ingest Locaction - the path where the person loading the dataset can find the data.
Archive Location - the path the dataset should be archived in.
External Contact Should contain the name and e-mail address of the person to contact when the data loading has been completed, this is not necessarily the contact loaded in CODIAC.
Remote URL If the dataset is linked to a remote website or a non-CODIAC webpage that URL should be entered here. If the dataset is loaded into CODIAC leave this field blank. This URL is used by the Master Data Set List.
 


Master Data Set List
Viewing Dataset Information
Datasets By Title
Clicking All Datasets By Title in the left frame displays all the datasets in the project. To sort the list click the heading titles (Dataset Title, Date etc.). This view is helpful because it is possible a dataset is not linked to a category and will therefore not show up when viewing datasets by category.
Datasets By Category
Clicking All Data Categories in the left frame displays all the datasets sorted by project. To view datasets in just one category click the cateogy names on the left, this will also show any sub-categories.
Add/Edit Dataset Information
Clicking the next to a dataset in the By Title view or the By Category view will produce a new window to edit the corresponding dataset.
Date Posted This date should correspond to the date JOSS posted the dataset to the master list but can be editted freely. The today link places today's date in the Date Posted field. The blank link places 0000-00-00 in the Date Posted field and will show up blank in the public view.
Storm Id The CODIAC storm_id for the dataset. If the dataset does not have a storm_id simply leave the field blank, do NOT enter 99.999, doing so may cause problems in the public view.
Doc URL The URL of the dataset's readme file or documentation.
Remote URL If the dataset is a remote website enter the URL here. If the link in the ML should point to the CODIAC dss? page leave this blank and be sure the Storm Id is correct.
Dataset Status
Select New, Updated, or In Progress to reflect the status of the dataset. The associated icon or text will then replace the date in the dataset list.
Hide Dataset Clicking this checkbox hides the dataset from the public Master List and will by hilighted with a dark gray in the editor.
Add/Edit Project Information
Currently, the only way to add a dataset to the ML/DLN system is through the Master List. Click Master Data Set List at the top and then click Add New Project on the left. This will bring up a new window which will allow you to add a new project.
Project Name The Project Name used in CODIAC (e.g. IHOP_2002).
Full Name The Full Project Name (e.g. Internatial H20 Project 2002).
Storm Id Prefix The storm_id prefix used for the project's datasets (e.g. 77). Used to determine the next storm_id when adding new datasets.
Logo Source The source of the logo to place in the public view. Refer to the footnotes of the Add/Edit Project form for more details. This field can be left blank.
Left CSS Source The source of the cascading style sheet to use in the left frame of the public view. Refer to the footnotes of the Add/Edit Project form for more details. This field cannot be blank, if unsure the default css file is left_def.css.
Body CSS Source The source of the cascading style sheet to use in the body frame of the public view. Refer to the footnotes of the Add/Edit Project form for more details. This field cannot be blank, if unsure the default css file is body_def.css.
Active This checkbox should probably always be checked. If the project is flagged inactive (the checkbox is un-checked) the project will be hidden from the system. The database will have to be updated using SQL to reset the project to active. This could be considered a bug.
Links to other webpages can be associated with a dataset to be placed under the Related Links section of the publi view. Refer to the Add/Edit Links portion of this help file.
