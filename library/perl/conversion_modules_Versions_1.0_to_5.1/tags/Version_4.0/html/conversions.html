Conversion Modules

The conversion modules are a collection of Perl modules that can be used for converting data into one of our standard formats. There is currently three types of formats that the modules can handle.

  • CLASS Sounding Format
  • QCF Surface Format
  • Station List Station Format

There are also several helper modules that assist in the conversions. They contain functions that are not specific to any conversion. They handle calculations, conversions of units, formatters, date/time modifiers, along with a set of constants that are used by them. A page of the equations and other information about the conversions, constants, etc. can be found on the Equations Page.

There are also a number of test scripts that are not listed with the Version 3 Modules list. These test the modules they are associated with to make sure they do what is expected in most cases. The test scripts are located in the same directory as the module it is testing.

DpgCalculations A collection of functions that can be included in a conversion for performing calculations of values.
DpgConstants A set of constant values that are used be the calculations and conversions. They are well defined constants such as the acceleration due to gravity and the gas constant for dry air.
DpgConversions A collection of functions that can convert values from one unit or measurement to another.
DpgDate A collection of functions that relate to dates and times. This includes formatters and converters.
Record A basic definition of a record for data. It contains functionality on how to handle dates, time, and station information. It does not do anything with data. It is meant to be a super class for more specific records.
RecordMap A data structure that is used to hold Record instances.
TestModule A collection of functions used for testing the modules. It is similar to the JUnit TestCase and Assert classes that are used in java. They are the super class for the conversion testing scripts (not listed.)
